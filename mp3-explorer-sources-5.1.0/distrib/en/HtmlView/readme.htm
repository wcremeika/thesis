3. Identified issues


- !!! New (4.1) !!! You may encounter a software crash on a corrupted file while scanning (it could happen). In this case, to activate the trace mode, just set in the MP3 EXPLORER.INI file located in Windows directory, in the Setup section, a "Debug" entry to 1 as follow :
...
[Setup]
Debug=1
...
 This will create a debug.txt in the [ mp3 - explorer ] directory in which you will see all files processed, and the last one should be the one with the problem. Don't forget to remove this line for a normal use of the software.

- !!! Important !!! On some platform, the address bar does not appear and the wonderful animate icon is on the left side of the window. It's a known problem reported by Microsoft due to an outdated version of the comctl32.dll. The version of this DLL should be 4.71 or upper. To get an updated version go to :
http://support.microsoft.com/download/support/mslfiles/Com32upd.exe

- Due to an access conflict on the file, the current playing file is not retrieved by the scanning process. To refresh the list or the cache file, it's better to stop your player.

- The command Enqueue works only with player that support the feature at the shell level. With some player as Sonique for example the Play command do the same as Enqueue if you select the 'Append to playlist' option.

5. FAQ


Q1. Problem with multiple selections play.

A1. Multiple selections are handle with a temporary playlist file named '~playlst.m3u'
in the TEMP directory. Verify 'm3u' is correctly associated to your player in the Shell.

7. Uninstall

Since version 4.2, the distribution provides an uninstaller

To remove completely [ mp3 - explorer ] manually from your system, you should delete the following files :

  • mp3 explorer.exe
  • readme.htm
  • *.edf

from the installation directory which is by default :
c:\Program Files\mp3 explorer
and the directory itself.

You should also delete in the Windows directory the file :

  • mp3 explorer.ini

If you were using cache files, you can delete all mp3cache.mex located in your MP3 file directories.

Since v2.0.1, no information is saved in the registry base, so only deleting files will leave your system totally clean.
